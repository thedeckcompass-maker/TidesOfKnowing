# Tides of Knowing: New Content Intake Workflow

This document outlines the workflow for integrating new website URLs into the Tides of Knowing Social Intelligence System. This process ensures that new content is systematically processed, categorized, and strategically promoted to maximize its impact and align with commercial objectives.

## New Content Intake Workflow Steps

When Leigh adds a new website URL, the following steps must be followed:

1.  **Add to Inventory:** Add the new URL to the `TOK_CONTENT_INVENTORY.md` [1].
2.  **Initial Status:** Mark the `Status` of the new entry as `Unprocessed` [1].
3.  **Assign Content Type:** Assign the appropriate `Content Type` (e.g., Article, Field Note) [1].
4.  **Provisional Objective:** Assign a provisional `Primary Objective` [1].
5.  **Scan on Instruction:** Scan the URL once when explicitly instructed by Leigh [1].
6.  **Create Source Card:** Create a detailed Source Card for the content using the `SOURCE_CARD_TEMPLATE.md` [1].
7.  **Create Angle Bank:** Generate an Angle Bank for the content using the `ANGLE_BANK_TEMPLATE.md` [1].
8.  **Launch Push Assets:** Create launch-push social assets only if strategically and commercially justified [1].
9.  **Evergreen Rotation:** After any initial launch push, transition the content to `Evergreen Rotation` status [1].
10. **Rescan Policy:** Do not rescan the live page unless its status in the `TOK_CONTENT_INVENTORY.md` is explicitly marked `Updated, Rescan Needed` [1].

## Rule for New Content

> New content should receive a preliminary social push where commercially or strategically justified, then enter evergreen rotation [1].

This rule ensures that new content receives appropriate initial visibility while integrating it into the long-term content strategy.
