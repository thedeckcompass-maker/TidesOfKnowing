# Tides of Knowing: Daily Operating Workflow

This document outlines the precise steps for the daily operation of the Tides of Knowing Social Intelligence System. Adherence to this workflow ensures efficient, signal-driven content production that aligns with strategic objectives and credit management rules.

## Daily Workflow Principle

Daily automation should continue even when Leigh does not provide manual signals. Manual signals improve the daily run, but the absence of signals should not stop ordinary scheduled production.

## Daily Workflow Modes

The daily workflow has two modes:

### Mode 1: Autopilot Daily Run

Use this mode when Leigh provides no new manual signals.

In Autopilot Daily Run, Manus may:

1.  Select one eligible source item from the existing inventory.
2.  Use only existing source cards and angle banks.
3.  Use current scores, next eligible dates, unused angles, platform-selection rules, and the social rotation ledger.
4.  Generate the minimum useful set of complete publishable social assets justified by the objective.
5.  Create matching image briefs where relevant.
6.  Update the social rotation ledger.
7.  Update the visual calendar or calendar-ready file when available.
8.  Stop.

In Autopilot Daily Run, Manus must not:

*   Rescan website pages.
*   Process new URLs.
*   Create new source cards unless explicitly instructed.
*   Create new angle banks unless explicitly instructed.
*   Rescore content without a signal.
*   Recommend extra development unless there is an existing recorded signal.
*   Use paid overage or add-on credits.

### Mode 2: Signal-Enhanced Daily Run

Use this mode when Leigh supplies one or more manual signals.

Signal prompt remains:

```
GSC signal:
Website signal:
Social signal:
Audience question:
Revenue signal:
Priority override:
```

In Signal-Enhanced Daily Run, Manus may:

1.  Use the supplied signals to adjust priority or select the next best content item.
2.  Update the relevant inventory row or audience signal log where justified.
3.  Generate complete publishable assets.
4.  Create matching image briefs where relevant.
5.  Update the ledger.
6.  Update the visual calendar or calendar-ready file when available.
7.  Stop.

Any recommendation for extra scanning, development, batching, new workflows, or expanded processing must use the exception-approval format from `CREDIT_AND_TASK_CONTROL_RULES.md`.

## Daily Operating Rules

*   **No Blanket-Posting:** Do not blanket-post content across all platforms. Every asset must have a strategic reason for its existence and platform choice [2].
*   **No Forced Daily Posting:** Do not force daily posting to every platform. Output should be driven by objectives, not by a need to fill a quota [2].
*   **Flexible Platform Mix:** Do not artificially limit output to one platform if multiple platforms genuinely serve the objective. The system should identify the best platform mix [2].
*   **Reddit Content Constraint:** Do not generate Reddit content unless there is a real Reddit context, a specific question, or an approved listening opportunity. Reddit is for careful participation, not automation [2].
*   **Reddit Approval:** All Reddit drafts require manual approval before use [2].
*   **Rescan Policy:** Do not rescan live website pages unless the content inventory status for that page explicitly states `Updated, Rescan Needed` [2].
