# Tides of Knowing Social Intelligence System: Manus Transfer Boundary Rules

This document defines the operational boundaries for the Tides of Knowing Social Intelligence System within Manus and outlines the strategy for its transfer to an external, canonical production environment. It clarifies what Manus has been used for, what it should not be used for going forward, and the intended future home for various tasks.

## 1. What Manus Has Been Used to Build

Manus has been instrumental in:

*   Establishing the core folder structure and foundational documents.
*   Defining and refining all strategic and operating rules.
*   Creating and populating the master content inventory with initial URLs.
*   Developing comprehensive templates for Source Cards, Angle Banks, Social Assets, and Image Briefs.
*   Implementing detailed workflow documents for daily operations, weekly intelligence, and new content intake.
*   Installing and integrating the `TOK_EDITORIAL_VOICE_AND_PLATFORM_ADAPTATION_GUIDE.md`.
*   Conducting controlled, end-to-end workflow tests for a single content item, validating the system's design and interlinking.
*   Creating a minimal, clickable visual calendar shell.
*   Generating an export readiness audit and system completion checklist.

## 2. What Manus Should Not Be Used For Going Forward

After the system's export, Manus should **not** be used for:

*   **Bulk production of Source Cards:** For the remaining 269+ inventory items.
*   **Bulk production of Angle Banks:** For the remaining 269+ inventory items.
*   **Bulk generation of Social Assets:** For all content items.
*   **Bulk generation of Image Briefs:** For all content items.
*   **Dynamic Visual Calendar Management:** The current calendar is a static Markdown file; dynamic updates and advanced features are for the external system.
*   **Ongoing Social Media Scheduling:** This should be handled by dedicated external tools.
*   **Long-term content storage:** The canonical repository will be external.
*   **Any task that incurs significant paid overage or add-on credits without explicit authorization.**

## 3. What Must Be Moved to the External Canonical System

All files within the `Tides of Knowing Social Intelligence System` directory must be exported and moved to the external canonical system. This includes:

*   All Markdown documents (rules, templates, inventory, ledger, calendar, assets, briefs, logs).
*   The entire folder structure.

## 4. What Can Safely Remain in Manus as a Reference Copy

The Manus environment can safely retain a reference copy of the system files for historical context or debugging purposes. However, this copy will **not** be the canonical production repository.

## 5. What Tasks May Still Be Done in Manus Only With Explicit Authorisation from Leigh

*   **System Debugging or Minor Patches:** If an issue is identified that requires Manus's unique capabilities for diagnosis or a small, targeted fix.
*   **New System Feature Prototyping:** For testing entirely new workflow components or templates in a controlled environment before integrating them into the external system.
*   **Ad-hoc, high-complexity content generation:** For a single, highly strategic content piece that requires Manus's advanced reasoning capabilities, with explicit credit approval.

## 6. What Tasks Should Move to ChatGPT

*   **Image Generation:** Using the generated Image Briefs as prompts.
*   **Initial Drafts of Long-Form Content:** For Substack expansions or articles, leveraging its creative writing capabilities.
*   **Content Refinement:** Polishing social asset copy, headlines, or CTAs based on the editorial voice guide.

## 7. What Tasks Should Move to Codex

*   **Automated Scripting:** For bulk generation of Source Cards, Angle Banks, Social Assets, and Image Briefs based on the established templates and rules.
*   **Dynamic Calendar Generation:** Developing scripts to parse the ledger and other files to create an interactive visual calendar.
*   **Data Processing and Analysis:** For automating the collection and analysis of audience signals, revenue data, and website analytics.

## 8. What Tasks Can Later Be Delegated to a VA

*   **Manual Data Entry:** Populating the `TOK_CONTENT_INVENTORY.md` with new URLs (after initial processing by Codex).
*   **Scheduling Social Assets:** Using external scheduling tools.
*   **Basic Content Review:** Proofreading and checking for adherence to simple rules.
*   **Image Sourcing/Curation:** Finding suitable stock images based on image briefs (if not AI-generated).

## 9. Credit-Control Warning

**Continued extensive use of Manus for bulk content production will incur significant credit usage.** The system has been designed for transfer to optimize cost-efficiency and scalability in an external environment.

## 10. Final Instruction

**Manus is NOT the canonical production repository after export.** All ongoing production, updates, and content management will occur in the designated external system. The Manus environment will serve as a historical record and a sandbox for authorized, specialized tasks only.
